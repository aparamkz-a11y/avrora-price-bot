# Телеграм-бот для прайс-листов AVRORA Steel

Бот получает Excel-прайс от поставщика, применяет наценку, заменяет контакты и возвращает два готовых файла — розница и опт.  
Работает в облаке Railway 24/7, доступ только с вашего Telegram ID.

---

## Что понадобится

| Что | Зачем | Сколько времени |
|-----|-------|-----------------|
| Telegram | Создать бота и получить токен | 5 мин |
| Аккаунт GitHub | Хранить файлы бота | 10 мин |
| Аккаунт Railway | Запустить бота в облаке | 10 мин |

**Итого: ~25 минут. Командная строка не нужна.**

## Шаг 1 — Получить токен бота через @BotFather

1. Откройте Telegram.
2. В строке поиска введите **@BotFather** и нажмите на него (синяя галочка — официальный бот).
3. Нажмите кнопку **START** (или напишите `/start`).
4. Напишите команду:
   ```
   /newbot
   ```
5. BotFather спросит: *"Alright, a new bot. How are we going to call it?"*  
   Введите **отображаемое имя**, например:
   ```
   AVRORA Price Bot
   ```
6. Затем спросит: *"Good. Now let's choose a username..."*  
   Введите **юзернейм** (латиница, должен заканчиваться на `bot`), например:
   ```
   avrora_price_bot
   ```
   Если имя занято — добавьте цифры: `avrora_price_2025_bot`.

7. BotFather ответит сообщением. Найдите строку вида:
   ```
   Use this token to access the HTTP API:
   7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   **Скопируйте весь токен** (от цифр до конца) и сохраните в блокнот. Это ваш `BOT_TOKEN`.

> ⚠️ Никому не показывайте токен — через него можно управлять ботом.

---

## Шаг 2 — Узнать свой Telegram ID

1. В поиске Telegram найдите **@userinfobot** (синяя галочка).
2. Нажмите **START**.
3. Бот немедленно ответит. Найдите строку:
   ```
   Id: 123456789
   ```
   **Запишите это число** — это ваш `ALLOWED_USER_IDS`.

> Если ботом будут пользоваться несколько человек — пусть каждый напишет @userinfobot и пришлёт вам свой ID. Их можно перечислить через запятую: `123456789,987654321`.

## Шаг 3 — Создать репозиторий на GitHub

### 3.1 Регистрация (если нет аккаунта)

1. Откройте браузер, перейдите на **github.com**.
2. Нажмите **Sign up** (правый верхний угол).
3. Введите email, пароль, имя пользователя.
4. Подтвердите email — придёт письмо от GitHub.

### 3.2 Создать новый репозиторий

1. После входа нажмите зелёную кнопку **New** (или плюс `+` → **New repository**).
2. Заполните форму:
   - **Repository name:** `avrora-price-bot`
   - **Description:** `Бот для обработки прайсов` (необязательно)
   - Выберите **Private** (приватный — файлы видите только вы)
   - Поставьте галочку **Add a README file**
3. Нажмите зелёную кнопку **Create repository**.

### 3.3 Загрузить файлы бота

Вам нужно загрузить **5 файлов** (не считая README.md, который уже там есть):

```
bot.py
process_supplier.py
suppliers_config.py
requirements.txt
Procfile
```

**Загрузка каждого файла:**

1. На странице репозитория нажмите кнопку **Add file** → **Upload files**.
2. Перетащите файлы или нажмите **choose your files** и выберите их на компьютере.
3. Внизу страницы нажмите зелёную кнопку **Commit changes**.

> Все 5 файлов можно загрузить за один раз — просто выберите их все сразу.

### 3.4 Проверить результат

После загрузки в репозитории должны быть видны все файлы:

```
📄 bot.py
📄 process_supplier.py
📄 suppliers_config.py
📄 requirements.txt
📄 Procfile
📄 README.md
```

## Шаг 4 — Запустить бота на Railway

### 4.1 Регистрация

1. Откройте браузер, перейдите на **railway.app**.
2. Нажмите **Login** → **Login with GitHub**.
3. GitHub попросит разрешение — нажмите **Authorize Railway**.
4. Railway создаст аккаунт автоматически.

> Railway даёт **$5 бесплатных кредитов** при регистрации + ~500 часов выполнения в месяц. Для одного лёгкого бота этого обычно хватает.

### 4.2 Создать новый проект

1. На главной странице Railway нажмите кнопку **New Project**.
2. Выберите **Deploy from GitHub repo**.
3. Railway попросит доступ к GitHub — нажмите **Configure GitHub App** → выберите ваш репозиторий `avrora-price-bot` → нажмите **Save**.
4. Вернитесь в Railway, выберите репозиторий `avrora-price-bot`.
5. Railway начнёт деплой — это займёт 1-2 минуты.

### 4.3 Добавить переменные окружения

После деплоя бот **ещё не работает** — нужно передать ему токен и ваш ID.

1. В Railway кликните на ваш проект.
2. Перейдите во вкладку **Variables** (иконка переменных в левом меню).
3. Нажмите кнопку **New Variable** и добавьте первую переменную:
   - **NAME:** `BOT_TOKEN`
   - **VALUE:** вставьте токен из Шага 1 (вида `7123456789:AAFxxx...`)
   - Нажмите **Add**

4. Снова нажмите **New Variable** и добавьте вторую:
   - **NAME:** `ALLOWED_USER_IDS`
   - **VALUE:** ваш Telegram ID из Шага 2 (вида `123456789`)
   - Нажмите **Add**

5. Railway автоматически **перезапустит бота** с новыми переменными.

### 4.4 Проверить, что бот запустился

1. Во вкладке **Deployments** должен быть статус **Active** (зелёный).
2. Если статус **Failed** (красный) — нажмите на деплой, откройте **View Logs** и найдите строку с ошибкой. Скопируйте её и напишите в чат с Клодом.

## Шаг 5 — Проверить работу бота

### 5.1 Первый запуск

1. В Telegram найдите вашего бота по юзернейму (тот, что придумали в Шаге 1).
2. Нажмите **START** или напишите:
   ```
   /start
   ```
3. Бот должен ответить приветственным сообщением. Если ответа нет — подождите 30 секунд и попробуйте ещё раз.

### 5.2 Отправить прайс Руфата

1. В чате с ботом нажмите **скрепку** (прикрепить файл) → **Файл**.
2. Выберите xlsx-файл от Руфата.
3. Бот ответит двумя файлами:
   - `[имя_файла]_розница.xlsx`
   - `[имя_файла]_опт.xlsx`

### 5.3 Отправить прайс Карагандинского

1. Аналогично — прикрепите xlsx-файл от Кавалерийского.
2. Бот также вернёт два файла с обработанными ценами и вашими контактами.

### 5.4 Если поставщик не распознан

Бот ответит сообщением вида:
```
❓ Не удалось определить поставщика.
Открой проект «AVRORA прайсы» в Claude, прикрепи этот файл 
и напиши: «новый поставщик, добавь конфиг».
```
Это нормально — просто новый поставщик ещё не настроен.

## Как добавить нового поставщика (например, MSK)

1. Бот пришлёт сообщение «не знаю формат».
2. Откройте Claude → проект **«AVRORA прайсы»**.
3. Прикрепите xlsx-файл нового поставщика.
4. Напишите: `новый поставщик, добавь конфиг`.
5. Я обновлю файл `process_supplier.py` с новой логикой.
6. Вы получите новую версию файла.

**Обновление файла на GitHub (без командной строки):**

1. Откройте ваш репозиторий на github.com.
2. Кликните на файл `process_supplier.py`.
3. Нажмите иконку **карандаша** (Edit this file) в правом верхнем углу.
4. Выделите весь текст (`Ctrl+A`), удалите, вставьте новый код.
5. Нажмите **Commit changes** → **Commit changes** (подтвердить).
6. Railway автоматически увидит изменение и перезапустит бота через ~1 минуту.

## Частые вопросы

**Бот не отвечает на /start**
→ Подождите 1-2 минуты после деплоя. Проверьте статус в Railway → Deployments.

**Ошибка "Unauthorized"**
→ Проверьте, правильно ли скопирован BOT_TOKEN в переменных Railway.

**Бот отвечает "У вас нет доступа"**
→ Проверьте ALLOWED_USER_IDS — должен быть ваш ID из @userinfobot.

**Railway показывает Failed**
→ Нажмите на деплой → View Logs → скопируйте последнюю строку с ошибкой → напишите в Claude.

**Хочу дать доступ коллеге**
→ Попросите коллегу написать @userinfobot, узнать свой ID. В Railway → Variables → ALLOWED_USER_IDS добавьте через запятую: `123456789,987654321`. Сохраните — бот перезапустится.
